Per-Email Processing

Step 1 — Match to a case (all local Python, no Claude call):

Tier 0: Conversation cache
Tier 1: RRID regex in subject/body
Tier 2: Case number substring
Tier 3: Match Keywords (whole-word)
Tier 4: Sender identifier

Open cases tried first. Ambiguous matches (2+ cases) → no match returned. Closed-case fallback only for Tiers 0-2.

Step 2 — Route into one of three paths:

Path A: Case Matched

Save email body as .txt to
Case Folder/Raw Documents/
Save all attachments (bytes)
to Raw Documents/
Append email_ingested event
to activity.jsonl
Skip Claude — log as case_matched

Path B: Possible Remy Request

Gate: sender domain in
REMY_SENDER_DOMAINS (bozzuto.com)
AND keyword match in subject/body
Claude classifier (claude-sonnet-4-5)
7 categories: breach_notice, nonrenewal,
warning_letter, settlement_agreement,
response_letter, dc_rent_complaint,
dc_breach_complaint
If enable_remy_invocation:
remy_runner.py parses form fields,
stages attachments, shells out to
Remy CLI → .docx output

Path C: Everything Else

Log to classifications.jsonl
with skip_reason: no_remy_signal
No Claude call, no action

CLI-Triggered Skills not in poll loop

Stage 2: Daily Run (instruction-driven)

python rocky.py --daily-run [RRID-XXXX]

Each case's _project/instructions.md tells Claude what to do. No hardcoded logic in Rocky.

Read _project/instructions.md from case folder
(skip case if missing)
Gather context: new raw files (with extracted text),
available subfolders, recent activity (48h)
One Claude call per case — instructions + context → JSON response:
{analysis, file_actions[], recommendations[]}
Execute file_actions: copy raw → target subfolder
Log daily_run event + any document_filed events
to activity.jsonl

Stage 3: Daily Case Digest

python rocky.py --daily-digest [RRID-XXXX] [--hours N]

Read activity.jsonl + master_file_index.json (last N hours)
Read latest Case Status Memorandum .docx
Claude generates per-case markdown:
What happened / Next steps / Upcoming dates
Write consolidated digest to
Daily Digests/YYYY-MM-DD.md
